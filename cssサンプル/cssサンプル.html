vertical-alignについて

inline、inline-block、table-cellである子要素に対し、親要素の縦方向のどの位置に配置するのかを決定する。
top,bottom以外は親要素に対する指定となる。ので親要素のfont-size、line-heightによって変化するので注意。top,bottomは行ボックスの上端、下端に対する設定となる。下記例では親ボックスには小さめのフォント、子には大きめのフォント、大き目の行間があり、行ボックスが大きくなっている。
基本は親要素(もしくはbodyとか)にfont-size、line-heightを設定してtop,bottom以外(つまり親要素に対する縦方向の指定)を設定するが、画像を同じ行に入れたりして行ボックスの高さが変化するときはtop、bottomで行ボックスに対する相対位置を決めるといいかも

height,widthの指定方法

基本的にはデフォルトでblock要素、inline-block要素においてwidthは親要素の幅いっぱい、heightは子要素の高さに合わせ成り行き。
inline要素においてwidthは子要素の幅に合わせ成り行き、heightも同様。
※pxはそのままなので例を省略！

width ％指定

親要素の幅(padding含まないcontentの幅)に対する割合い％で指定する。(親要素がbox-sizing:border-box;でもcontentの幅に対する割合となる。)合計が100%を超えないようにする。ただしデフォルトではbox-sizing:content-box;なのでコンテンツ幅が親要素いっぱいになる+paddingなので,親要素からはみ出てしまう。auto(初期値)だとpadding含みで親要素の幅いっぱいに調整される。これを解消するためにbox-sizing:border-box;を指定する(autoと同じ挙動になるはずだがレスポンシブ時は width:100%; と box-sizing:border-box; の方がいいかも)か、paddingとwidthを含め100%となるように指定するとよい。min-width,max-widthを指定しておくとレスポンシブ

30%

20%

auto

100% + content_box(初期値)

100% + border_box

width80% + padding 10%ずつ + content_box

height ％指定

親要素の高さ(padding含まず)に対する割合い％で指定する。合計が100%を超えないようにする。heightは初期値がautoなので、親要素には必ずheight:auto;以外の指定をしなければ子要素の％指定は不可。ただしwidthと同様デフォルトでpaddingを含む高さ指定なので,100%指定するとwidthが親要素の高さいっぱい+paddingで親要素からはみ出てしまう。これを解消するためにbox-sizing:border-box;を指定する。ただしwidthと同様にpaddingとheightを含め100%になるようにはできない(次で説明)。min-width,max-widthを指定しておくとレスポンシブ

margin,paddingの％指定は基本横方向のみ

基本横方向のみ％指定で使い、縦方向に使用すると前述のとおりレイアウトが崩れて見える。これは縦方向に％指定しているが、基準となるのは"親要素の高さ"ではなく"親要素の幅"を基準にしているためである。

padding: 25% 0;
横幅の25%ずつを上下にpadding
縦横比を保つことができる。

margin相殺について

縦方向のmarginのみ要素間でmargin相殺が起こる事は簡単だが子要素が親要素のmarginにも影響を与える場合があるので注意

誤りの例

こんな感じ。innerの中のdivにmargin-top:50px;をしているつもりだが親要素のinnerにmargin-top:50px;されてしまっている

margin-top: 50px;

対策１

親要素にpaddingする。親paddingからのmarginとして扱われるイメージ。これが自然な感じ。

margin-top: 50px;

対策２

親要素にborderを入れる。borderからのmarginとして扱われるイメージ。heroとか画面いっぱいに広げたい要素内にmarginを設定したいならborder色を背景色と一緒にしてこれがいいかも

margin-top: 50px;

対策３

親要素にoverflow:hidden;を入れる。これもheroとか画面いっぱいに広げたい要素内にmarginを設定したいならいいかも

margin-top: 50px;

対策４

なんらかのブロック要素を入れて邪魔をする。根本対策ではない
